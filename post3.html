Github Style Guide

01 May 2020

Part of the rubric talks about how to format the commentary for updates to the file repositories on Github. Some of the stuff they specify makes sense. The problem is that my mind doesn't work in the same boxes theirs apparently do, particularly when it comes to the types of updates I make. If I'm doing near-wholesale changes to the content of my webpage, it is NOT a "feature". When I upload new stylesheets, or changes to the styles, how am I supposed to categorize it?
